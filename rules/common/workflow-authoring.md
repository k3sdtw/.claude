# Workflow 스크립트 작성 규칙

> Workflow 도구로 실행하는 인라인 스크립트의 **단일 안전 기준**.
> 원칙 하나만 지키면 샌드박스 제약 위반(결정성·args·meta 등)이 **구조적으로** 사라진다 —
> 실패 모드마다 가드를 덧붙이는 두더지 잡기를 끝낸다.

## 단일 원칙: 로직은 고정, 데이터만 변한다

Workflow 스크립트는 매 실행마다 새로 쓰지 않는다. **로직(meta·schema·orchestration·return)은 스킬이 제공하는 보일러플레이트를 그대로 복사**하고, 모델은 **명시적으로 표시된 데이터 블록 1곳만** 채운다.

데이터 블록에는 **순수 리터럴만** 넣는다:

- 허용: 문자열, 숫자, 불리언, 배열·객체 리터럴, 보간 없는 정적 백틱 문자열
- 금지: 함수 호출, 표현식, 조건·반복, 그 외 모든 로직

> 모델이 코드를 쓰지 않으면 코드 버그도 없다. 이 규칙은 **닫힌 규칙**이라, 새 샌드박스 제약이 생겨도
> "리터럴만, 로직 금지"가 이미 포함하므로 문서를 고칠 필요가 없다.

## 샌드박스 제약 (위 원칙이 자동으로 커버하는 것들)

| 제약 | 이유 | 데이터 블록에 절대 넣지 말 것 |
|------|------|------------------------------|
| 비결정성 함수 금지 | resume 시 캐시된 결과 replay와 어긋남 | `Date.now()` · `new Date()` · `Math.random()` |
| `meta`는 순수 리터럴 | 정적으로 파싱됨 | 변수 · 함수 호출 · 스프레드 · 템플릿 보간 |
| FS·Node API 없음 | 샌드박스 격리 | `fs.*` · `require` · `process` |

## 데이터를 스크립트에 넣는 법

스크립트는 **FS 접근이 없어** 외부 파일에서 데이터를 읽지 못한다. 채널은 둘:

1. **인라인 리터럴 (권장)**: 데이터를 스크립트 본문의 표시된 블록에 직접 박는다. args 전송 계층이 없어 가장 견고하다.
2. **`args` (신뢰가 검증된 경우만)**: Workflow 호출의 `args`에 **실제 JSON 값**으로 전달한다 — 문자열로 직렬화하면 스크립트에서 문자열로 도달해 `args.x.map`이 `undefined`로 터진다. 확증 전엔 1번을 쓴다.

## 시각·유니크 값이 필요하면

- **타임스탬프**: 스크립트 안에서 찍지 말고, **워크플로우가 반환된 뒤** 메인 루프에서 Bash `date -u +%Y-%m-%dT%H:%M:%SZ`로 찍는다.
- **유니크 값**: `Math.random()` 대신 `map((x, i) => ...)`의 인덱스 `i`나 이미 존재하는 식별자(area·name 등)를 쓴다.

## 단일 fan-out이면 Workflow 대신 Task

barrier 없는 단순 병렬(N개 fan-out → 집계)에는 Workflow 스크립트가 과하다. **Task 도구 병렬**(한 메시지에 여러 Task)이 더 단순하고 샌드박스 제약에서 완전히 자유롭다. 구조화 출력은 프롬프트에 JSON 형식을 명시해 강제한다.

Workflow는 **phase 간 순차 barrier · resume · in-code 결정적 시퀀싱**이 실제로 필요할 때만 쓴다.
