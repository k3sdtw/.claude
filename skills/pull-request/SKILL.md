# Pull Request Skill

GitHub Pull Request를 일관된 형식으로 생성합니다.

## PR 작성 원칙

1. **제목**: 70자 이내, 변경의 핵심을 담음
2. **본문**: 한국어로 작성, 필요한 정보만 포함
3. **리뷰어 관점**: 변경 이유와 영향 범위 명확히

## 템플릿 구조

```markdown
## 개요
{이 PR이 해결하는 문제 또는 추가하는 기능 1-2문장}

## 주요 변경사항
- {변경사항 1}
- {변경사항 2}
- {변경사항 3}

## 테스트
- [ ] E2E 테스트 추가/수정
- [ ] 로컬에서 테스트 완료
- [ ] 기존 테스트 통과 확인

## 참고사항
{리뷰어가 알아야 할 컨텍스트, 제약사항, 후속 작업 등}
```

## PR 유형별 가이드

### 기능 추가 (feat)
- 개요: 어떤 기능이 추가되는지
- 주요 변경사항: 추가된 API, 엔티티, Use Case 등
- 테스트: 새 기능에 대한 E2E 테스트

### 버그 수정 (fix)
- 개요: 어떤 버그를 수정하는지
- 주요 변경사항: 수정된 로직
- 테스트: 버그 재현 방지 테스트

### 리팩토링 (refactor)
- 개요: 리팩토링 목적
- 주요 변경사항: 구조/패턴 변경 내용
- 테스트: 기존 동작 보장 확인

## 체크리스트

PR 생성 전 확인:
- [ ] 커밋 메시지가 컨벤션을 따르는가?
- [ ] Biome 린트가 통과하는가?
- [ ] 불필요한 파일이 포함되지 않았는가?
- [ ] 민감 정보가 노출되지 않았는가?

## Commands

- `/pr` - Pull Request 생성
